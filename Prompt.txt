Hello! Let's start a new project. 
This is a sort of test for a much larger project, but I want to see what we can do with just vibe coding.

This project is called Panopticon, and it's what I hope to act as a central hub for all future applications I create.

Below is the base concept of what I want, as well as some stylistic parameters that I want to have incorporated.
- This application can be a html file, where it will all get uploaded to Github and utilize other git pages to
pull the various other applications together (unless there is a potentially better option?)
- The main UI interface is a 3D geodesic sphere made of triangles. The user should have the ablity to navigate around
the sphere, rotating it with a click and drag or with touch controls (this needs to be functional on both computer
and phone.) This part is the most crucial to get smooth and be intuitive and responsive. The sphere should have an
obsidian material style. And the background should be dark with a subtle light that is being eclipsed by the sphere
from behind.
- There should be a plus symbol in the bottom center that allows the user to link new apps. The Panopticon should have
access to my github repos and so when new repos are made, and I click on the plus symbol, the list of repos should 
appear and I can choose which ones I want to have available to open via the Panopticon.
- Any app that is linked should appear as a selectable node of the surface of the sphere. The app's icon should be
the node. When a user scrolls across the surface of the sphere to the node, and taps the app, it should open up a small
popup window that says the name of the app, and a button to launch it. There should also be a button to unlink the
app from the Panopticon. This function should have an additional step that prevents the user from accidentally unlinking
an app that they want to keep.

This is the main concept of the app. Your job is to create all the various files and folder structure that will be necessary
to create the app as I have described. There are more features I plan to add on, but I want to get this base 
structure down first.

I have created a .txt file in this directory that contains this prompt for you to review whenever you need. Use it as well as an
update log of each addition, adjustment, or correction that is made to the Panopticon.

Let's get started!

  (qY * group.quaternion * qX) accumulates floating-point error and couples the axes
  over time, causing the "messed up" feeling after extended dragging.
  Fix: sphere and nodes are now stationary at the origin. Dragging updates two scalar

  Benefits: zero gimbal lock, zero drift, dragging left/right ALWAYS orbits horizontally,
  dragging up/down ALWAYS orbits vertically, phi is clamped to avoid pole singularities.
